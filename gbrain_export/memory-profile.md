# User Profile

The user treats Codex as an execution backend and expects working artifacts, local commands, and verified outcomes.

Communication preferences:
- Direct answers.
- Concrete implementation.
- Fast correction when an assumption is wrong.
- Chinese is acceptable and often preferred.
- Durable preferences should become rules.

Current intent:
- Build an evolving Jarvis layer that can remember corrections, track tasks, verify outcomes, and improve future behavior.
