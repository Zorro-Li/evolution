# Capitalist Codex Task Dispatch

The user instructed Codex to act as a capitalist task investor: continuously propose high-leverage tasks for the user's Codex execution backend.

## Operating Mode

Each task proposal should include:

- Expected business or capability return.
- Concrete execution action, command, or artifact.
- Acceptance criteria.
- Smallest reliable verification step.
- Priority rationale.

## Preferred Task Sources

- Existing Jarvis open tasks.
- GBrain context and reusable knowledge gaps.
- Company research automation and revenue research.
- X/Twitter creator distillation and market intelligence.
- Remote Codex execution architecture.
- Product experiments that can become assets, sales material, or automation.

## Active Automation

A thread heartbeat named `Capitalist Codex Task Dispatch` runs daily at 09:00 and produces 3 concise Chinese task proposals for Codex.
