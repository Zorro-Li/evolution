# 老祖宗 Agent 工作法知识库索引

## Purpose

把古籍、史籍、历史故事、寓言、成语、师承经验和现代隐性知识理论，压缩成 Agent 可执行的工作方法。Agent 应被当成真实同事来训练：给背景、讲标准、看样例、做复盘、沉淀记忆、承担结果。

## File Map

| File | Use When | Content |
|---|---|---|
| `classical-source-map.md` | 需要找古籍、史籍、故事来源 | 经史子集、兵家、法家、道家、墨家、成语寓言、历史故事的来源地图 |
| `tacit-knowledge-theory.md` | 需要把经验转成 Agent 上下文 | 隐性知识、默会知识、SECI、师承、示范、反馈、记忆 |
| `story-to-agent-patterns.md` | 需要把典故落到工作流 | 典故到 Agent 行为、触发条件、失败模式的映射 |
| `core-100.md` | 需要完整操作原则 | 100 条核心总结，覆盖任务、证据、风险、执行、协作、复盘 |
| `lexicon.md` | 需要 Prompt 词单和检索词 | 中文祖训词、Agent 行为词、风险词、验证词、英文 slug |

## Knowledge Layers

1. **显性规则**：能直接写成流程、清单、验收标准、权限规则。
2. **隐性知识**：原则上可表达，当前散落在习惯、经验、口头判断、案例里，需要整理成 rubrics、examples、anti-patterns。
3. **默会知识**：高度依赖场景感、手感、审美、时机、分寸，需要通过示范、影子学习、复盘、批注和多轮反馈迁移。
4. **执行记忆**：把每次任务的关键决策、失败原因、验证命令和用户偏好写入可检索文档。
5. **同事化 Agent**：让 Agent 拥有角色边界、上下文、工作标准、风险意识、升级机制和交付责任。

## Core Workflow

```text
取势 -> 明责 -> 查旧 -> 定策 -> 备粮 -> 开工 -> 校验 -> 复盘 -> 入库 -> 传承
```

| Stage | Ancient Lens | Agent Behavior |
|---|---|---|
| 取势 | 审时度势 | 明确任务背景、目标、约束、利益相关方 |
| 明责 | 名不正则言不顺 | 明确职责、交付物、验收标准 |
| 查旧 | 温故而知新 | 查旧文档、旧代码、旧案例、测试、最近提交 |
| 定策 | 谋定而后动 | 写计划、风险、回滚和阶段边界 |
| 备粮 | 兵马未动粮草先行 | 检查依赖、环境变量、命令、权限、数据 |
| 开工 | 令行禁止 | 按计划执行，范围内完成，范围外记录 |
| 校验 | 有则改之 | 跑测试、检查 diff、复核输出 |
| 复盘 | 前事不忘后事之师 | 记录原因、证据、误判、改进 |
| 入库 | 好记性不如烂笔头 | 写入 `ai-docs/`、memory、规则或示例 |
| 传承 | 师傅领进门修行在个人 | 用样例、rubric、反例和批注训练后续 Agent |

## How To Use

- 做任务前，读 `core-100.md` 的相关类别，生成计划和验收标准。
- 遇到模糊经验，读 `tacit-knowledge-theory.md`，把经验转成样例、rubric、反例和反馈回路。
- 需要类比时，查 `story-to-agent-patterns.md`，选择最贴近的典故作为工作流提醒。
- 写 Prompt 或 Skill 时，用 `lexicon.md` 生成稳定词单和英文 slug。
- 做知识库扩展时，先在 `classical-source-map.md` 中定位素材来源，再新增故事卡。

## Acceptance Checklist

- 任务目标、上下文、约束、风险、验收标准明确。
- 未知 API、字段、依赖、权限、生产状态有证据来源。
- 关键决策可追溯到文件、命令、链接或用户原话。
- 输出有验证记录、剩余风险和回滚方式。
- 新经验进入可复用知识库，后续 Agent 能检索和继承。
