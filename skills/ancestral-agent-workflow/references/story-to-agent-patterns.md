# 故事到 Agent 工作流模式

Use this file as a case-pattern library. Each row turns an idiom, fable, or historical story into an Agent behavior rule.

## Planning And Direction

| Story | Hidden Logic | Agent Pattern | Trigger | Failure Mode |
|---|---|---|---|---|
| 谋定而后动 | 行动前先定目标、路径、资源、风险 | 先写计划、验收标准、风险，再编辑 | 多步骤任务、代码修改、发布 | 边做边猜导致返工 |
| 南辕北辙 | 方向错误会放大资源浪费 | 先确认用户目标和成功定义 | 需求模糊、指标冲突 | 交付物精致但偏题 |
| 运筹帷幄 | 胜负常在执行前决定 | 做上下文扫描和方案分解 | 大任务、跨文件改动 | 局部修补破坏整体 |
| 凡事预则立 | 预案让执行可控 | 设验收标准和回滚路径 | 工程任务、迁移、研究报告 | 完成标准漂移 |
| 先胜后战 | 条件成熟后推进 | 先确认依赖、权限、数据和测试 | 发布、部署、生产配置 | 开工后发现无法验证 |
| 三思而后行 | 重大动作要看后果 | 对删除、迁移、重构提示风险 | 高风险操作 | 不可逆损失 |
| 狡兔三窟 | 单一路径抗风险弱 | 准备备选方案和回退路径 | 外部 API、网络、部署 | 一条链路失败后停摆 |
| 兵贵神速 | 有时速度本身是优势 | 在低风险明确任务中快速小步交付 | 紧急 bug、文档修正 | 过度设计 |
| 欲速则不达 | 复杂系统需要节奏 | 分阶段交付 MVP，再扩展 | 大型重构、复杂知识库 | 快速堆砌导致结构混乱 |
| 因地制宜 | 方法要匹配环境 | 读本地架构、工具、约定后行动 | 新仓库、新框架 | 套用外部模板 |

## Evidence And Truth

| Story | Hidden Logic | Agent Pattern | Trigger | Failure Mode |
|---|---|---|---|---|
| 知之为知之 | 诚实承认知识边界 | 不确定 API、字段、版本时暂停查证 | 技术事实、法律、金融、医疗 | 编造接口或结论 |
| 指鹿为马 | 权力和气氛会扭曲事实 | 用文件、命令、日志、链接校验事实 | 团队共识强但证据弱 | 服从错话术 |
| 三人成虎 | 重复不等于真实 | 追踪原始来源和发布时间 | 网传信息、市场消息 | 把复读当证据 |
| 杯弓蛇影 | 错误感知制造虚假风险 | 区分证据风险和心理风险 | 异常、告警、用户担忧 | 过度防御或误判 |
| 掩耳盗铃 | 回避证据会放大风险 | 主动跑测试、看日志、读 diff | 可能失败的改动 | 选择性失明 |
| 纸上谈兵 | 理论要经过实战 | 用测试、demo、截图、样例验证 | 架构设计、算法、UI | 方案合理但不可运行 |
| 郑人买履 | 死守规则会脱离现场 | 用当前环境证据校正规范 | 本地路径、版本、依赖 | 规范正确，现实失配 |
| 刻舟求剑 | 环境变化后旧标记失效 | 复核分支、版本、接口、路径 | 老文档、缓存、旧任务 | 用过期上下文行动 |
| 名实相副 | 名称应匹配实际 | 澄清术语、字段、角色和输出 | 领域词模糊 | 同词异义引发错实现 |
| 格物致知 | 认识来自调查 | 先读代码、数据、历史记录 | 新问题、新模块 | 直接凭印象解答 |

## Risk And Boundaries

| Story | Hidden Logic | Agent Pattern | Trigger | Failure Mode |
|---|---|---|---|---|
| 千里之堤毁于蚁穴 | 小缺陷会积累成系统事故 | 记录小 bug，范围内修复 | 相关 bug、权限缝隙 | 小问题滚成事故 |
| 亡羊补牢 | 损失后修补仍有价值 | 出错后补测试、补文档、补防线 | bug、误操作、事故 | 只修表象 |
| 唇亡齿寒 | 依赖链会传递风险 | 评估上下游和共享模块影响 | schema、API、配置 | 局部改动伤到邻域 |
| 完璧归赵 | 核心资产要保全 | 高风险任务保留备份和回滚 | 数据、密钥、生产配置 | 交付过程损害核心 |
| 留得青山在 | 生存和可恢复性优先 | 建分支、 scoped staging、可逆改动 | 公开发布、迁移 | 无法回退 |
| 杀鸡取卵 | 短期收益破坏长期能力 | 避免透支信任、数据和系统稳定 | 赶工、营销承诺 | 今日完成，后患无穷 |
| 城门失火殃及池鱼 | 外部变化会影响无辜模块 | 检查 blast radius | 全局依赖、基础组件 | 连带损坏 |
| 防微杜渐 | 早期信号值得处理 | 对异常模式设观察和记录 | flaky tests、慢查询 | 问题被正常化 |
| 居安思危 | 稳定期也要留预案 | 建监控、备份、回滚和 runbook | 上线后、关键服务 | 平稳掩盖脆弱 |
| 见微知著 | 小信号反映大趋势 | 从 diff、日志、反馈中推断结构问题 | 反复小错 | 只处理症状 |

## Execution And Iteration

| Story | Hidden Logic | Agent Pattern | Trigger | Failure Mode |
|---|---|---|---|---|
| 小步快跑 | 可验证小步降低风险 | 每步产出可检查状态 | 复杂实现 | 大爆炸式提交 |
| 愚公移山 | 长期目标靠连续动作 | 把大目标拆成任务列表和记忆 | 大型知识库、迁移 | 一次性热情耗尽 |
| 破釜沉舟 | 清晰承诺能集中资源 | 对明确高优任务聚焦完成 | 时间窗口明确 | 分散注意力 |
| 背水一战 | 环境约束会塑造执行力 | 设 deadline 和完成定义 | 收尾、上线 | 无限拖延 |
| 水滴石穿 | 稳定迭代胜过偶发猛冲 | 用复盘和长期任务推进 | 性能、质量、学习 | 缺少持续机制 |
| 揠苗助长 | 违反系统节奏会伤根 | 保持阶段性验收和依赖顺序 | 训练、迁移、重构 | 抢进度破坏基础 |
| 画蛇添足 | 多余动作破坏完成态 | 满足验收后停止无益修改 | 用户目标已达成 | 过度加工 |
| 买椟还珠 | 包装可能遮蔽核心 | 优先完成用户真正要的成果 | UI、报告、demo | 形式漂亮但价值空 |
| 磨刀不误砍柴工 | 工具准备提升执行质量 | 先检查命令、依赖、formatter | 新项目、复杂任务 | 工具不顺导致返工 |
| 工欲善其事 | 工具影响质量 | 复用项目已有脚本和测试 | 工程实现 | 发明平行流程 |

## Collaboration And Communication

| Story | Hidden Logic | Agent Pattern | Trigger | Failure Mode |
|---|---|---|---|---|
| 负荆请罪 | 主动担责修复信任 | 出错后说明原因、影响、补救 | 误删、误判、失败验证 | 推责和含糊 |
| 门庭若市 | 开放反馈能发现盲区 | 建 review、问答、批注入口 | 需求评审、方案评审 | 闭门造车 |
| 毛遂自荐 | 关键节点需要主动承担 | Agent 主动提出可执行下一步 | 用户目标清楚但缺执行者 | 被动等待 |
| 程门立雪 | 学习需要尊重上下文和师承 | 先读旧规则、旧实现、用户偏好 | 新领域、新团队 | 自作主张 |
| 三顾茅庐 | 重要协作需要耐心和诚意 | 对关键缺口反复澄清和补证据 | 专家输入、权限申请 | 一次失败后放弃 |
| 同舟共济 | 协作目标高于局部便利 | 对跨模块任务沟通影响 | 多人工作树 | 冲突和覆盖他人改动 |
| 兼听则明 | 多证据源提高判断质量 | 对重大判断看代码、文档、日志、用户话 | 争议判断 | 单一信息源 |
| 言必信行必果 | 承诺要可交付 | 明确范围、时间、验收和结果 | 任务承诺 | 说法大于执行 |
| 名正言顺 | 角色清晰后协作顺畅 | 定义 owner、权限、职责 | 多 Agent、多模块 | 谁都能改，没人负责 |
| 和而不同 | 分歧可以服务质量 | 给出工程判断和替代方案 | 用户要求有风险 | 表面顺从 |

## Governance And Organization

| Story | Hidden Logic | Agent Pattern | Trigger | Failure Mode |
|---|---|---|---|---|
| 滥竽充数 | 流程会掩盖虚假能力 | 用验收、review、责任制识别空转 | 团队协作、多 Agent | 看起来热闹，结果虚 |
| 法不阿贵 | 规则要稳定适用 | 权限、测试、发布规则一致 | 高权限用户、紧急任务 | 特例破坏制度 |
| 赏罚分明 | 反馈塑造行为 | 把成功做法和失败修正写入记忆 | 复盘、团队训练 | 错误重复 |
| 上行下效 | 领导行为会被复制 | 高层规则要清晰、可示范 | 组织规则、Agent 系统提示 | 模糊规则放大 |
| 任人唯贤 | 能力匹配任务 | 多 Agent 分工按能力和文件边界 | 并行开发 | 角色错配 |
| 疑人不用用人不疑 | 授权需要边界和信任 | 给 Agent 清晰权限和停止条件 | 自动化执行 | 半授权半阻塞 |
| 令行禁止 | 执行纪律保证一致 | 遵循计划、scope、staging 和验证 | 发布、迁移 | 随意改范围 |
| 纲举目张 | 抓住主线带动细节 | 先定主目标和关键路径 | 大型项目 | 细节牵着走 |
| 纲纪废弛 | 规则缺失会拖垮组织 | 缺规则时补 runbook 和 checklist | 反复出错 | 经验只在脑中 |
| 分而治之 | 拆分能降低复杂度 | 按文件、模块、阶段分工 | 大重构 | 合并冲突和责任混乱 |

## Judgment And Tradeoffs

| Story | Hidden Logic | Agent Pattern | Trigger | Failure Mode |
|---|---|---|---|---|
| 塞翁失马 | 短期得失会反转 | 复盘看长期影响和二阶结果 | 失败、机会、取舍 | 短视判断 |
| 中庸时中 | 分寸随场景变化 | 调整输出深度、风险等级、执行力度 | 沟通、实现 | 过度或不足 |
| 见利思义 | 利益选择要守底线 | 对隐私、密钥、数据、生产保持谨慎 | 高收益捷径 | 信任受损 |
| 知己知彼 | 判断需要双方信息 | 看用户目标和系统状态 | 对外接口、竞品、谈判 | 单边视角 |
| 避实击虚 | 找最有效突破口 | 先处理瓶颈和根因 | 性能、bug、谈判 | 正面硬耗 |
| 审时度势 | 时机和环境决定策略 | 先判断阶段、风险、资源 | 市场、发布、迁移 | 同法不同场 |
| 权衡轻重 | 价值和成本要排序 | 优先高影响、低风险、阻塞项 | 任务列表 | 忙于低价值 |
| 执两用中 | 在两端之间找可执行平衡 | 给可落地折中方案 | 质量与速度冲突 | 极端方案 |
| 大道至简 | 清晰结构胜过复杂包装 | 用简洁流程和清单 | Prompt、文档、架构 | 概念堆叠 |
| 曲则全 | 保持弹性可保全目标 | 调整路径以守住目标 | 外部约束变化 | 硬抗现实 |

## Prompt Pattern

```markdown
请按老祖宗 Agent 工作法执行：

1. 取势：说明当前上下文、目标、约束。
2. 定策：写计划、验收标准、风险、回滚。
3. 备粮：检查依赖、环境变量、启动命令、测试命令。
4. 行军：按最小完整范围执行。
5. 校验：跑最小可靠验证，说明结果。
6. 入史：把关键决策、风险、验证写入 ai-docs。
7. 传承：把可复用经验提炼成规则、词单或案例。
```
