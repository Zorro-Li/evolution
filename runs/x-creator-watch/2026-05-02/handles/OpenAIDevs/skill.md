# x-creator-openaidevs

Use this skill to track `@OpenAIDevs`, official OpenAI developer updates, Codex, OpenAI Platform, slash-command vocabulary, Codex Pets, `/pet`, `/hatch`, skill installation, service status, developer campaigns, and user-generated workflow demos.

## Capture Workflow

1. Open `https://x.com/OpenAIDevs` in Safari through Computer Use.
2. Capture profile metadata, banner text, pinned post, current product commands, API/platform posts, service-status links, contests, and retweeted user demos.
3. Classify posts as `codex_command`, `platform_update`, `api_example`, `skill_distribution`, `developer_campaign`, `contest_terms`, `community_demo`, `service_status`, or `interface_pattern`.
4. Preserve source URL, visible metrics, command names, quoted accounts, media clues, prizes, legal terms, and workflow steps.
5. Convert each signal into a Jarvis product or operating-system UI experiment.

## Extraction Fields

- Command vocabulary: `/pet`, `/hatch`, `/goal`, future slash commands, skill invocation syntax.
- Platform surface: Codex, OpenAI Platform, developers.openai.com, status.openai.com.
- Skill distribution: `@skill-installer`, skill names, install syntax, workflow prompts.
- Campaign mechanics: submission ask, artifact type, prize, dates, eligibility, legal terms.
- UX patterns: background agent state, peripheral status, desktop companions, progress visibility, user delight.
- Adoption evidence: retweeted user demos, languages, account graph, media type, engagement.

## Output Format

Return:

| Field | Content |
|---|---|
| Source | post URL and capture surface |
| Developer signal | command/platform/skill/campaign/status/interface signal |
| Evidence | visible text, metrics, command, quoted account, media clue, or legal term |
| Product implication | how OpenAI wants developers to use or share the capability |
| Jarvis implication | local command, skill, status UI, demo loop, or automation test |
| Watch graph | accounts, commands, skills, docs, and platform links |

## Failure Modes

- Treating playful Codex posts as low-signal novelty and missing official command vocabulary.
- Saving contests without extracting artifact loop, prize, legal terms, and growth mechanic.
- Missing retweeted users as early workflow evidence.
- Ignoring skill-installer syntax in community examples.
- Tracking API posts and Codex posts in separate buckets when the profile explicitly combines Codex and the OpenAI Platform.
