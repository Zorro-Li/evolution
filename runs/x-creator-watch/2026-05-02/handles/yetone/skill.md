# Skill Distillation: `@yetone`

## Skill Name

`x-creator-yetone`

## Purpose

Track yetone, `avante.nvim`, AI chat app design, provider orchestration, memory UX, Codex `/goal`, local-first agent clients, and craft-focused indie AI software.

## Trigger Conditions

Use this skill when the user asks about:

- yetone or `avante.nvim`
- AI chat app product design and memory UX
- coding-agent goal persistence and editor-native agents
- local-first agent clients
- AI product taste and indie software craft
- provider orchestration across OpenAI, Anthropic, Gemini, and custom providers

## Workflow

1. Capture the current visible `@yetone` profile batch through Safari / Computer Use.
2. Classify each item:
   - own product launch
   - profile project graph
   - coding-agent primitive
   - AI chat memory/provider orchestration
   - local-first agent client
   - product craft signal
   - creator/demo tooling
3. Extract:
   - product or tool named
   - underlying workflow
   - architecture implication
   - taste principle
   - Jarvis/local-agent relevance
   - watch-graph additions
4. Convert the signal into local action:
   - add agent-loop primitive to Jarvis notes
   - inspect linked project pages or GitHub when needed
   - turn provider orchestration into architecture questions
   - preserve polished indie tools as UX references
   - compare editor-native agent patterns with local Codex workflows
5. Write a brief ranking signals by Jarvis value, product taste, and implementation relevance.

## Failure Modes

- Reading quote posts as simple likes and missing the taste signal.
- Ignoring profile project links while over-weighting a single post.
- Saving `/goal` as a product mention without mapping it to persistence.
- Treating AI chat apps as generic chat surfaces and missing memory/provider orchestration.
