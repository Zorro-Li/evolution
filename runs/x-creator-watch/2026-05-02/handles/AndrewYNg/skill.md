# Skill Draft: AI Education And Workflow Framework Watcher

Use this skill when tracking AI educators and framework builders such as `@AndrewYNg`, DeepLearning.AI, practical AI course launches, AI developer conferences, and posts that turn AI trends into operational concepts.

## Inputs

- X handle
- visible posts and pinned posts
- course titles
- named frameworks
- partner tools or instructors
- link cards and event pages
- engagement metrics when visible

## Workflow

1. Capture the visible account batch from Safari / Computer Use.
2. Split posts into:
   - course launch
   - framework or concept definition
   - developer workflow shift
   - tool ecosystem signal
   - event or community signal
3. Normalize each item into:
   - source URL
   - concept name
   - target audience
   - promised skill or workflow
   - linked tool or partner
   - local Jarvis relevance
4. Convert each signal into local action:
   - add a concept to the watch graph
   - create a checklist for a Jarvis workflow
   - test a tool or technique locally
   - track the partner account or event page
5. Produce a brief that ranks signals by execution value and education-market demand.

## Output Template

```md
# AI Education Framework Brief

## Current Thesis
<what the creator is teaching the market>

## Evidence Table
| URL | Type | Concept | Audience | Local Jarvis action |
|---|---|---|---|---|

## Frameworks To Save
- <framework>: <definition>

## Tool/Event Watch Graph
- <tool or event>: <reason>
```

## Failure Modes

- Treating course launches as simple promotion.
- Missing named concepts that become durable vocabulary.
- Saving the course topic without extracting the workflow.
- Ignoring partner tools and instructors that reveal the ecosystem map.
