# x-creator-manxis

Use this skill to track `@Manxis_lab`, ManXis Agent, next-generation AI agents, general-purpose agentic intelligence, foundation-model agent systems, and applied-intelligence launch signals.

## Capture Workflow

1. Open `https://x.com/Manxis_lab` in Safari through Computer Use.
2. Capture profile metadata, banner text, follower count, post count, and any new links.
3. If posts appear, classify each item as `launch_signal`, `demo`, `product_claim`, `research_claim`, `team_signal`, `docs_link`, `github_link`, or `waitlist`.
4. Preserve URL, visible metrics, media, links, and quoted accounts.
5. Convert each new signal into a Jarvis agent-architecture implication.

## Extraction Fields

- Product surface: ManXis Agent, docs, waitlist, demo, GitHub, API.
- Agent capability: planning, tool use, memory, autonomy, browser/computer use, multi-agent coordination, task completion.
- Research framing: foundation models, applied intelligence, general-purpose agentic intelligence.
- Evidence type: profile-only, launch thread, demo video, benchmark, paper, customer workflow.

## Output Format

Return:

| Field | Content |
|---|---|
| Source | profile or post URL |
| Signal type | profile-only, launch, demo, docs, research, or product claim |
| Evidence | visible text, link, media, metric, or account relation |
| Agent implication | architecture or capability meaning |
| Verification target | local test, external doc, demo, or future watch query |

## Failure Modes

- Over-interpreting the account before original posts exist.
- Missing first-launch posts because the current account has no timeline corpus.
- Treating broad AGI-adjacent wording as evidence without demos, docs, benchmarks, or product links.
