# x-creator-portkey

Use this skill to track `@PortkeyAI`, Portkey, AI Gateway, production AI control plane, reliability layer, autonomous-agent security, Palo Alto Networks acquisition, Prisma AIRS integration, and agent framework integrations such as Hermes Agent.

## Capture Workflow

1. Open `https://x.com/PortkeyAI` in Safari through Computer Use.
2. Capture profile positioning, banner text, acquisition/funding state, and follower/post counts.
3. Classify posts as `funding_signal`, `acquisition_signal`, `control_plane_thesis`, `security_integration`, `agent_integration`, `investor_signal`, or `ecosystem_retweet`.
4. Preserve source URL, metrics, link cards, quoted accounts, acquirer/investor names, and integration names.
5. Convert the signal into an enterprise AI infrastructure implication.

## Extraction Fields

- Infrastructure thesis: production AI, control plane, reliability layer, governance, routing.
- Security layer: Palo Alto Networks, Prisma AIRS, autonomous-agent security, enterprise confidence.
- Gateway mechanics: AI Gateway, model routing, policy, observability, evaluation, failover.
- Capital/acquirer graph: Series A, Elevation, Lightspeed, Palo Alto Networks.
- Agent graph: Hermes Agent, Nous Research, agent frameworks that need production controls.

## Output Format

Return:

| Field | Content |
|---|---|
| Source | post URL and capture surface |
| Infrastructure claim | gateway/control-plane/security/reliability claim |
| Evidence | visible text, metric, acquirer/investor/integration, or link card |
| Enterprise implication | buyer, risk, governance, reliability, or deployment implication |
| Jarvis implication | local gateway, policy, logging, failover, or security test |
| Watch graph | accounts, tools, companies, and integration names |
