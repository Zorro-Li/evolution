# Skill Distillation: @yifan_zhang_

## Purpose

Use Yifan Zhang as a research watch source for LLM reasoning, RL, pretraining, positional encoding, RoPE, and model architecture commentary.

## Workflow

1. Capture profile metadata and current pinned technical post.
2. Classify posts by position encoding, attention, RoPE, multimodal architecture, LLM reasoning, RL, pretraining, benchmark reaction, or research link.
3. Extract paper links, blog links, ArXiv IDs, named methods, and quantitative benchmark/price claims.
4. Treat bookmark-heavy technical posts as durable research references.
5. Convert architecture claims into Jarvis research notes with model-component implications.

## Output Template

```markdown
## Yifan Research Signal

- Source:
- Method or paper:
- Claim:
- Evidence:
- Model component:
- Practical implication:
- Follow-up source:
```

## Failure Modes

- Treating a short prediction as consensus.
- Missing the difference between LLM core architecture and multimodal components.
- Saving model benchmark numbers without source and model version.
