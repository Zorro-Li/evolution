# x-creator-vercel

Use this skill to track `@vercel`, Vercel, agentic infrastructure, AI apps, AI agents, Vercel AI SDK, v0, production deployment, AI Accelerator, Ship events, OAuth risk, environment variable risk, npm supply-chain posture, and platform-security trust signals.

## Capture Workflow

1. Open `https://x.com/vercel` in Safari through Computer Use.
2. Capture profile positioning, verification, follower/post counts, hiring signals, pinned post, and visible product/event/security posts.
3. Classify posts as `agentic_infrastructure`, `production_customer_story`, `ai_accelerator`, `product_launch`, `security_incident`, `security_guidance`, `partner_signal`, or `hiring_signal`.
4. Preserve source URL, date, visible metrics, link cards, media clues, partner accounts, affected platform surface, and admin action requested.
5. Convert the signal into an AI application infrastructure implication and a local Jarvis platform/security check.

## Extraction Fields

- Platform thesis: agentic infrastructure, apps, agents, frontend cloud, deployment, v0, AI SDK.
- Production proof: customers shipping agents and apps, Ship talks, workshops, case studies, AI Accelerator winners.
- Partner graph: AWS, OpenAI, Anthropic, GitHub, Microsoft, npm, SocketSecurity.
- Security layer: OAuth apps, Google Workspace admin review, environment variables, MFA, npm package integrity, incident disclosure.
- GTM signal: enterprise sales, developer ecosystem, product marketing security, product marketing v0, GRC, pricing, field engineering.

## Output Format

Return:

| Field | Content |
|---|---|
| Source | post URL and capture surface |
| Platform claim | app/agent/deployment/security/production claim |
| Evidence | visible text, metrics, link card, partner graph, or hiring clue |
| Buyer implication | developer, platform, security, enterprise, or admin implication |
| Jarvis implication | local deployment, OAuth review, secret handling, app-agent workflow, or browser automation test |
| Watch graph | accounts, products, partners, and incident surfaces |

## Failure Modes

- Treating Vercel as frontend hosting only and missing the agentic infrastructure pivot.
- Saving event posts without extracting customer production workflows.
- Reading AI Accelerator posts as marketing copy and missing vertical-agent demand.
- Tracking the security incident without converting it into OAuth, secret-management, and admin-review controls.
- Ignoring hiring signals that reveal enterprise, security, v0, and ecosystem priorities.
