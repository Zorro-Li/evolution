# x-creator-portkeyai

Use this skill to apply Portkey-style production AI control-plane thinking to Jarvis, LLM apps, gateways, and autonomous agent systems.

## Trigger

Use when the task involves production AI, AI gateways, agent security, governance, observability, prompt management, budget controls, guardrails, approvals, or enterprise agent deployment.

## Inputs

- Agents/tools.
- Model/provider routes.
- Actor identity.
- Sensitive action list.
- Approval policy.
- Logging and audit requirements.
- Cost/rate limits.

## Steps

1. Map all model and tool calls through a single control plane.
2. Define policy, approvals, and blocked actions.
3. Add request, actor, tool, model, cost, and outcome logs.
4. Add gateway routing and fallback behavior.
5. Add spend, rate, and error dashboards.
6. Add security review for privileged agent actions.
7. Convert stable policies into Jarvis rules.

## Output

Return a production AI control-plane plan with policies, audit records, fallback paths, cost controls, and verification commands.

## Validation

- Every privileged action has approval and audit state.
- Every provider route has fallback behavior.
- Every cost rule has observable counters.
