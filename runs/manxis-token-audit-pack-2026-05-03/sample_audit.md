## 5. Optimization Plan

| Patch | Owner | Expected impact | Risk |
|---|---|---|---|
| Add usage capture wrapper | ManXis | Enables token/cost proof | Depends on Codex CLI usage visibility. |
| Cache source ledgers | ManXis | Reduces repeated web context | Needs freshness timestamp. |
| Deterministic report validator | ManXis | Reduces final cleanup tokens | Validator must preserve references. |
| Split source collection and report writing | ManXis | Allows reuse across companies | More pipeline code. |
