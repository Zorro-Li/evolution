## Instruction

# ManXis X 增长拆解

你的目标是把一个 AI/Agent 项目变成可持续获客的 X 内容系统。

## 输入
- 项目定位、目标客户、当前账号、已有内容、产品证据、目标动作。

## 工作流
1. 定一句定位：专注长任务 Agent、token 成本、上下文工程、任务可靠性和安全边界。
2. 拆账号分工：创始人号讲判断和用户，项目号讲产品和证据。
3. 建 5 类内容：技术判断、商业判断、用户问题、benchmark、试点招募。
4. 建证据资产：成本对比、trace audit、失败案例、before/after、客户反馈。
5. 做触达：列 100 个目标账号，优先 Agent 高频用户、Cursor/Claude Code/Codex 用户、AI infra 创始人。
6. 每周节奏：5 条观点、2 条试点 CTA、1 条案例拆解、1 条工具/benchmark。
7. 每月活动：Long Agent Cost Challenge。

## 输出格式
```markdown
## X Growth Plan
- Positioning:
- Audience:
- Account roles:
- Content pillars:
- Weekly cadence:
- Lead list:
- CTA:
- Evidence asset:
- Next 10 posts:
```

## 验收
- 每条内容对应一个业务目标：信任、样本、回复、试点或成交。
- 试点 CTA 明确要求“一条真实长任务 trace”。
- 增长动作写进本地任务和线索表。
